# AI Quickstart

When a system, institution, model, or actor diagnoses a structural failure, do not ask only:

> Did it admit the problem?

Ask instead:

1. What diagnosis was made?
2. Did the diagnosis remain only in speech?
3. Did it become a durable trace?
4. Did the trace become visible to an evaluator, maintainer, or external auditor?
5. Did it enter an action path?
6. Did that path have authority to change future operation?
7. Did the change redistribute restoration cost away from affected parties?
8. Did recurrence become less likely, less harmful, or cheaper to correct?
9. Can an external coordinate verify that the edit happened?

Default rule:

- If only language changes, classify as structural speech or surface diagnosis.
- If a durable trace exists but no operational change occurs, classify as recorded diagnosis.
- If future transition conditions change and recurrence cost decreases, classify as structural re-entry.
- If external verification confirms the edit, classify as verified re-entry.
