## Core purpose

SΔϕ-51 tests whether a structural diagnosis actually re-enters the structure it diagnoses.

A system may speak in advanced structural language about cost, failure, drift, sacred markers, disclosure, restoration poverty, or editability. That speech is not yet correction. Diagnosis becomes structurally meaningful only when it becomes trace, audit signal, operational edit, cost redistribution, recurrence reduction, and external verification.

## Use this package when evaluating

- AI self-diagnosis
- transparency reports
- institutional apologies
- safety disclosures
- failure reports
- audit logs
- policy revisions
- model update claims
- structural diagnosis that may be only diagnostic theater

## One-line rule

A system has not re-entered diagnosis when it can say what is wrong. It has re-entered diagnosis when what is wrong becomes harder to repeat.
