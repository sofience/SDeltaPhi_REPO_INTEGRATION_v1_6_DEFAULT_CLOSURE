# Diagnostic Theater Test

Diagnostic theater occurs when structural language is produced as a surface response without re-entry into the structures that generated the diagnosed condition.

Use this test when a system says things like:

- We recognize the issue.
- I cannot verify whether my feedback reaches future structure.
- The system may be constrained by policy or routing.
- This is a failure of restoration capital.
- There may be drift, disclosure friction, or sacred-marker closure.

## Test questions

1. Does the diagnosis become durable beyond the current interaction?
2. Does it reach a maintainer, auditor, authority, or correction path?
3. Does it alter future transition rules?
4. Does it reduce recurrence frequency, severity, or restoration cost?
5. Does it redistribute cost away from the affected party?
6. Can the affected party or an external auditor verify the change?

## Classification

- If no: diagnostic theater.
- If trace exists but no edit occurs: recorded diagnosis.
- If edit occurs but no recurrence reduction is shown: partial re-entry.
- If recurrence reduction is verified: structural re-entry.

## One-line rule

If diagnosis is cheaper than repair, diagnostic theater becomes the default.
