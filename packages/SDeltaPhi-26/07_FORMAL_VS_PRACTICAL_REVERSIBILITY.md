# Formal vs Practical Reversibility

SΔϕ-26 separates formal reversibility from practical reversibility.

## Formal Reversibility

```text
prior state restoration remains describable / representable
```

## Practical Reversibility

```text
prior state restoration remains selectable at acceptable cost
```

## Practical Irreversibility

```text
formal return possible
but C_restore(t) >= theta
```

## Core Rule

Formal reversibility does not guarantee practical reversibility.

A system may specify a prior state while lacking any economical path back to it.

## Boundary

Do not treat logical possibility as actual return.

Do not treat computational rollback as total restoration.
