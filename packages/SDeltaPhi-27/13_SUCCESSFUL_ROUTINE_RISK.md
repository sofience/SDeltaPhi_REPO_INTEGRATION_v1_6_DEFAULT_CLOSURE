# Successful Routine Risk

A path can become dangerous not because it is malfunctioning, but because it is functioning too smoothly.

## Risk

```text
smooth continuation
-> exception visibility suppressed
-> low friction mistaken for truth
-> operational success mistaken for universal adequacy
-> cheap operation hardens into authority
```

## Diagnostic question

Does the system monitor only catastrophic failures, or also successful routines?

## Rule

Any system that values editability must monitor successful routines, not only breakdowns.
