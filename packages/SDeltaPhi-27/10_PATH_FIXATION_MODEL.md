# Path Fixation Model

Path fixation occurs when repeated selection structurally privileges a path and makes revision progressively harder.

```text
R(P)↑ -> F(P)↑
```

## Mechanism

Repeated low-cost operation reorganizes the system around the selected path.

```text
skills
records
incentives
expectations
interpretive habits
tooling
authority structures
```

begin to presuppose the path.

## Consequence

The cost of returning is no longer one reversal step.

It includes undoing everything organized under the cheap continuation regime.

```text
F(P)↑ -> C_rest(P)↑
```
